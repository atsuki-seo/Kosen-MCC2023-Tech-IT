# 記述問題 (Short answer) 作問例

情報工学科の授業（アルゴリズム、データ構造等）を想定したShort answer問題の例。回答が複数通りありうる場合は完全一致の列挙ではなく、ワイルドカードを組み合わせて表記揺れを吸収する設計パターンを示す。

---

## 例1: 基礎（データ構造の用語）

### 問題文
後入れ先出し（LIFO: Last In, First Out）の原則で要素を管理するデータ構造の名称を、日本語のカタカナで一語で答えよ。

### 正解バリエーション
| 回答 | fraction | フィードバック |
|------|----------|---------------|
| スタック | 100% | 正解です。スタックはLIFO（Last In, First Out）の原則でデータを管理するデータ構造です。 |
| `*` | 0% | 不正解です。LIFOの原則で動作するデータ構造は「スタック」です。push（積む）とpop（取り出す）操作で要素を管理します。 |

### Moodle設定
- `<usecase>0</usecase>`（大文字小文字を区別しない）

### 設計意図
- 難易度: 基礎
- 観点: データ構造の基本用語（LIFO/FIFO等）の正確な知識を確認
- 日本語カタカナの回答なので表記揺れが少なく、完全一致のみで十分
- catch-allフィードバックで正解と簡単な説明を提示

---

## 例2: 標準（計算量の記法）

### 問題文
要素数 n の整列済み配列に対する二分探索の最悪時間計算量を、ビッグオー記法で答えよ。括弧には丸括弧 `()` を使い、対数の底は省略してよい（例: `O(n)`）。

### 正解バリエーション

Moodleは上から順に評価するため、ワイルドカードの広いパターンは下に配置する。

| 回答 | fraction | フィードバック | マッチ方式 |
|------|----------|---------------|-----------|
| `O(*log*n*)` | 100% | 正解です。二分探索は毎回探索範囲を半分にするため、最悪時間計算量は O(log n) です。 | ワイルドカード: O() で囲み log と n を含む回答すべてにマッチ（スペース有無、括弧、底の表記を一括吸収） |
| `*log*n*` | 50% | 惜しいですが、ビッグオー記法 O() で囲んで書く必要があります。正解は O(log n) です。 | ワイルドカード: log と n を含むが O() がない回答に部分点 |
| `*` | 0% | 不正解です。二分探索の最悪時間計算量は O(log n) です。探索範囲が毎回半分になるため、最大 ⌈log₂(n+1)⌉ 回の比較で探索が完了します。 | catch-all |

### Moodle設定
- `<usecase>0</usecase>`（O と o の両方を受け入れ）

### 設計意図
- 難易度: 標準
- ビッグオー記法の正しい書き方と計算量の理解を同時に確認
- ワイルドカード `O(*log*n*)` で表記揺れ（`O(log n)`, `O(logn)`, `O(log(n))`, `O(log2 n)`, `O(log₂n)` 等）を一括吸収し、完全一致の列挙を不要にする
- 部分点（50%）で「log n」のみの回答に対応し、O() 記法の不足を指摘

## 例3: 応用（アルゴリズムの性質）

### 問題文
クイックソートの平均時間計算量は O(n log n) であるが、最悪時間計算量はこれより悪化する。クイックソートの最悪時間計算量をビッグオー記法で答えよ。括弧には丸括弧 `()` を使うこと（例: `O(n)`）。

### 正解バリエーション

Moodleは上から順に評価するため、ワイルドカードの広いパターンは下に配置する。

| 回答 | fraction | フィードバック | マッチ方式 |
|------|----------|---------------|-----------|
| `O(*n*2*)` | 100% | 正解です。クイックソートはピボットの選択が偏ると毎回1要素しか分割できず、最悪 O(n²) になります。 | ワイルドカード: O() で囲み n と 2 を含む回答にマッチ（`O(n^2)`, `O(n²)`, `O(n2)` 等を一括吸収） |
| `O(*n*n*)` | 100% | 正解です。クイックソートの最悪時間計算量は O(n²) です。 | ワイルドカード: `O(n*n)` のような乗算表記にマッチ |
| `*n*2*` | 50% | 惜しいですが、ビッグオー記法 O() で囲んで書く必要があります。正解は O(n²) です。 | ワイルドカード: n と 2 を含むが O() がない回答に部分点 |
| `*` | 0% | 不正解です。クイックソートの最悪時間計算量は O(n²) です。ピボットが常に最小値または最大値に選ばれる場合、分割が偏り n-1 回の再帰が必要になります。 | catch-all |

### Moodle設定
- `<usecase>0</usecase>`

### 設計意図
- 難易度: 応用
- 平均計算量と最悪計算量の違いを正確に理解しているかを確認
- 問題文で平均計算量を提示することで、「同じ O(n log n) 」と混同する学生を判別
- ワイルドカードでべき乗の表記揺れ（`^2`, `²`, `*n`, `2`）を一括吸収
