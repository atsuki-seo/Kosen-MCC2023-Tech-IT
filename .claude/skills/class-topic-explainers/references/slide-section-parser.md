# スライド MD 節抽出ガイド

Marp スライド MD から「節」の境界と本文範囲を特定する方法をまとめる。節 = 1 Issue の単位なので、ここでの抽出精度が Issue の質に直結する。

## 全体アルゴリズム

```
1. スライド MD を YAML フロントマターと本文に分離
2. 本文を「---」で区切ってスライド配列にする
3. 目次スライドを探す
   - あり: 節が複数ある回とみなす → (4)
   - なし: 節が1個の回とみなす → (7)
4. 章区切りスライドを特定
5. 目次項目と章区切りを突合
   - 一致: 節名・本文範囲を確定 → 処理成功
   - 不一致: フォーマット異常 → 当該回スキップ（警告）
6. 各節について本文範囲を切り出して返す
7. 節が1個の回: 表紙から節名を抽出、表紙以外の全スライドを本文範囲にする
```

節が複数か1個かの判定は**目次スライドの有無**で行う。class-slides の仕様（`references/slide-structure-guide.md` § 4）で「節が1個しかない回: 目次スライド自体を省略する」と定められているため、目次がない = 節が1個と一意に対応する。

## 目次スライドの特定

### パターン

以下の条件をすべて満たすスライドを「目次スライド」と判定する:

- スライドの最上位見出しが `# 目次`（完全一致、階層は `#` = h1）
- 見出し以降の本文が箇条書きのみ（`- ` または `* ` で始まる行）で構成されている
- 通常は表紙スライドの直後に配置される（スライド配列で 1〜2 番目）

### 目次項目の抽出

目次スライド本文の箇条書き項目（`- ` または `* ` のあとの文字列）を順序保持のリストとして取り出す。

**例:**

```markdown
---

# 目次

- 授業概要と3本柱
- 評価方法と学修単位
- 開発環境の構築

---
```

→ 目次項目リスト: `["授業概要と3本柱", "評価方法と学修単位", "開発環境の構築"]`

### 表記ゆれの扱い

`# 目次` 以外の見出し表記（`## 目次`, `# 今回の内容`, `# もくじ` 等）は**目次として認識しない**。class-slides が生成する目次見出しは `# 目次` で固定されているため、ゆれを受け入れると判定が不安定になる。認識できない場合は「目次なし」として扱い、節が1個の回の判定に進む。

## 章区切りスライドの特定

### パターン

以下の条件をすべて満たすスライドを「章区切りスライド」と判定する:

- スライドの最上位見出しが `#`（h1）で1つだけ
- 見出し以降の本文が**空、または空行・コメントのみ**
- Marp ディレクティブ（`<!-- _class: ... -->` 等）は本文なし判定に影響しない（ディレクティブはコメントアウトされているため）

**例（章区切りスライド）:**

```markdown
---

# 授業概要と3本柱

---
```

### 見出しのテキスト抽出

章区切りスライドの見出しから `# ` を除去したテキストをそのまま取得する。

**例:** `# 授業概要と3本柱` → 節名 `授業概要と3本柱`

## 節本文の範囲特定

### 目次項目と章区切りの突合

目次項目リストの各項目と、スライド配列に現れる章区切りスライドの見出しテキストを**順序通り完全一致**で突き合わせる。

- 目次項目 N 個 = 章区切りスライド N 個、順序も一致 → 突合成功
- 個数が合わない、または同位置のテキストが一致しない → **フォーマット異常**

フォーマット異常の場合はフォールバックせず、当該回を抽出失敗としてスキップする（無理に節を推定すると誤マッピングで混乱を招くため）。

### 本文範囲の切り出し

各節について、**章区切りスライドの次のスライドから、次の章区切りスライドの直前まで**（最後の節の場合は最終スライドまで）を本文範囲とする。

```
スライド配列:
[0] 表紙
[1] 目次
[2] 章区切り: 節A ← 節Aの開始マーカー
[3] ...          ← 節Aの本文
[4] ...          ← 節Aの本文
[5] 章区切り: 節B ← 節Bの開始マーカー（節Aの本文はここで終わり）
[6] ...          ← 節Bの本文
...
```

節Aの本文範囲: スライド [3] 〜 [4]
節Bの本文範囲: スライド [6] 以降

### 本文範囲からの情報抽出

本文範囲の各スライドから以下を取得する:

- **最上位見出し**: そのスライドのキーコンセプト
- **本文全体**: Issue の「解説」セクションに使う原文

キーコンセプトリストは「この節で学ぶこと」セクションに、本文全体は「解説」セクション生成の入力になる。

## 節が1個しかない回の扱い

目次スライドが見つからない場合、節が1個の回とみなす。以下の方法で節名と本文範囲を決める。

### 節名の抽出

**表紙スライド**（通常スライド配列 [0]、`# [科目名] 講義資料` のようなタイトルと `## 第N回 XXX` のサブタイトルを含む）から、`## 第N回 XXX` の XXX 部分、または `第N回 XXX` 全体を節名とする。

**例の表紙スライド:**

```markdown
# ソフトウェア工学特論 講義資料

## 第1回 ガイダンス・環境構築

- 3本柱（Markdown / git・GitHub / プロジェクト管理）
- ...
```

→ 節名: `ガイダンス・環境構築`（`第1回` を除いたサブタイトル部分）

Issue タイトルは `[2026 ソフトウェア工学特論] 第1回 ガイダンス・環境構築` のように組み立てるため、節名に「第N回」を含めない形で保持しておく。

### 本文範囲

表紙スライド（`[0]`）を除く**全スライド**（`[1]` 以降）を本文範囲とする。目次・章区切りがないため、本文範囲の各スライドがそのまま解説対象になる。

## 抽出失敗時の判定

以下のいずれかに該当する場合、当該回の抽出を失敗とみなしてスキップする。他の回の処理は続行する:

1. 目次スライドがあるが、章区切りスライドが1つも見つからない
2. 目次項目数と章区切り数が一致しない
3. 目次項目と章区切り見出しのテキストが順序通りに一致しない
4. 表紙スライドがない、または `## 第N回 XXX` のパターンが見つからない

エラーメッセージは Step 7 の完了レポートに以下のように記載する:

```
⚠ スキップした節:
- 第N回: スライドから節を抽出できませんでした（目次と章区切りが不一致）
```

## GitHub アンカー生成

スライド MD の見出しへのリンクを作るため、GitHub の slugger 規則に従ってアンカー文字列を生成する。

### 変換ルール

1. 小文字化（日本語・全角文字はそのまま残る）
2. スペースを `-` に変換
3. 記号（`.`, `,`, `(`, `)`, `:`, `;`, `/`, `!`, `?`, `'`, `"`, `` ` ``, `#`, `*`, `+`, `=` 等）を除去
4. 連続する `-` を 1 つに圧縮
5. 先頭・末尾の `-` を削除

### 例

| 見出し | アンカー |
|-------|---------|
| `授業概要と3本柱` | `授業概要と3本柱` |
| `DFA の実装` | `dfa-の実装` |
| `Hello, World!` | `hello-world` |
| `評価方法 (学修単位)` | `評価方法-学修単位` |
| `第1回 ガイダンス・環境構築` | `第1回-ガイダンス環境構築` |

### 注意点

- **中点（`・`）の扱い**: GitHub の slugger は中点を削除する仕様。`ガイダンス・環境構築` → `ガイダンス環境構築`
- **長音記号（`ー`）はそのまま**: `データー` → `データー`
- **重複見出しのサフィックス**: 同一スライド MD 内で同じ見出しが複数あると `-1`, `-2` が付く。本スキルでは章区切り見出しが重複しない前提だが、もし重複するなら節抽出段階で警告する
- **完全性は保証できない**: GitHub 側の slugger 実装は非公開で、細かいエッジケースでアンカーが効かない場合がある。アンカーが効かなくてもファイル本体へのリンクとしては機能するため致命的ではない

実装時は生成したアンカーを直接 URL に埋め込むだけでよく、アンカー先の検証は Step で行わない（検証しようとすると GitHub API 呼び出しが増え、コストに見合わない）。
