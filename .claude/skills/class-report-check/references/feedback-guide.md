# フィードバックコメント記述ガイド

各提出者へのフィードバックコメントの書き方。

## コメント構成

フィードバックは以下の3部構成で記述する:

### 1. 良い点（1〜2文）
提出物の中で良くできている部分を具体的に指摘する。何が良いか・なぜ良いかを明確にする。

### 2. 改善点（1〜3文）
次回に向けて改善すべき点を具体的に指摘する。何をどう改善すればよいかの方向性を示す。

### 3. 次のアクション（1文）
次回のレポートや今後の学習に向けた具体的なアドバイスを1つ示す。

## 文体ルール

- 「です・ます」調で統一する
- 学生に直接語りかける形で書く（「あなたの〜」ではなく「〜の部分では」等、内容に焦点を当てる）
- 具体的なファイル名・セクション名・行番号を引用して指摘する

## NG表現

以下の表現は避ける:

| NG表現 | 代替表現 |
|--------|---------|
| 「〜が間違っています」 | 「〜の部分は〜のように修正すると、より正確になります」 |
| 「不十分です」 | 「〜についてさらに詳しく説明すると良いでしょう」 |
| 「理解できていません」 | 「〜の概念を復習し、〜の視点で再考してみてください」 |
| 「〜がダメです」 | 「〜について〜の観点を加えると改善されます」 |

## 文字数目安

- 全体で150〜300字程度
- 良い点: 50〜100字
- 改善点: 50〜150字
- 次のアクション: 30〜50字

## 特殊ケースの対応

### 満点（理想的が全観点）の場合
- 良い点を厚めに記述する（何が特に優れていたか）
- 改善点は「さらなる発展」の方向で記述する（「次のステップとして〜に挑戦してみてください」）

### 最低点（未到達が全観点）の場合
- 提出した事実を肯定する（「取り組んでいただいたことは評価しています」等）
- 改善点は最も優先度の高い1〜2点に絞る（多くの指摘は逆効果）
- 次のアクションは具体的で取り組みやすいものにする

### 未提出の場合
- フィードバックは生成しない
- 採点一覧に「未提出」と記録する
