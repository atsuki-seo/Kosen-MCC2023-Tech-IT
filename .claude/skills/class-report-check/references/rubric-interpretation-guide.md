# ルーブリック解釈ガイド

ルーブリックの段階判定に迷った場合の解釈ルール。

## 境界判定の原則

### 基本ルール

2つの段階の間で判断に迷う場合は、**低い方の段階を選ぶ**。理由: 自動採点では過大評価のリスクが過小評価のリスクより大きい（教員が後から加点するのは容易だが、減点は学生の不満を招く）。

### 複数基準がある観点

ルーブリックの1つの段階に複数の達成基準が記述されている場合:

- **すべて満たす** → その段階と判定
- **過半数を満たす** → その段階と判定（ただし標準点の下限寄りで点数をつける）
- **半数未満** → 1つ下の段階と判定

### 曖昧な表現への対応

ルーブリックに「適切に」「十分に」等の曖昧な修飾語がある場合:

- 課題の要件と照合して具体化する
- アンカー採点で確立した基準を参照して判断する
- 判断の根拠に「課題要件の[X]を満たしているため、ルーブリックの『適切に』に相当すると判断」と明記する

## 提出物形式別の注意点

### Markdownレポート

- 文書構成（見出し・段落・リスト）が適切か
- 内容の記述量と深さ
- 図表や参考文献の有無

### ソースコード

- コードが動作するか（構文エラーの有無を確認）
- コードの可読性（変数名、コメント、インデント）
- アルゴリズムの正確さと効率性
- テストコード・動作検証の有無

### PDF

- 文書全体の構成と体裁
- 内容の網羅性と深さ
- 図表の質と適切な引用

### 複数ファイルの統合評価

1提出者が複数ファイルを提出している場合:
- すべてのファイルを読み込んで総合的に評価する
- 「プログラム + レポート」の場合、プログラムの品質とレポートの品質を合わせて評価する
- ファイル間の整合性（レポートの記述とコードの内容が一致しているか）も考慮する
