# 履修者名簿・チーム構成ファイル フォーマット仕様

Step 0 のチーム構成ファイル読み込み、および Step 3 の履修者名簿読み込みで使用するフォーマット。

## 基本仕様

| 項目 | 仕様 |
|------|------|
| 形式 | CSV |
| 文字コード | UTF-8（BOM有無どちらも可）/ Shift-JIS を自動判定して受理 |
| ヘッダー行 | 1行目をヘッダーとみなす |
| 区切り文字 | カンマ (,) |

文字コード判定は以下の順で試行する:

1. UTF-8 BOM付き
2. UTF-8
3. Shift-JIS（Windows で Excel から CSV 保存した場合に多い）
4. いずれも失敗した場合はユーザーにエンコーディングを確認する

## 履修者名簿の列構成

### 必須列

- **学籍番号**: 学生の学籍番号
- **氏名**: 学生の氏名

その他の列（メールアドレス・所属・学年等）があっても構わないが、採点処理では使用しない。

### カラム名の揺らぎ対応

以下のいずれかのヘッダー名にマッチすれば、対応する論理カラムとして認識する:

| 論理カラム | 受理するヘッダー名 |
|-----------|-----------------|
| 学籍番号 | `学籍番号`, `学生番号`, `番号`, `No.`, `No`, `ID`, `学生ID` |
| 氏名 | `氏名`, `名前`, `学生氏名`, `学生名`, `Name` |

### 自動マッピングの流れ

1. CSV を読み込み、1行目のヘッダー値を取得する
2. 上記揺らぎパターンでヘッダーを論理カラムにマッピングする
3. **成功時**: 「学籍番号列: [ヘッダー名]、氏名列: [ヘッダー名] でマッピングしました」と表示し、`AskUserQuestion` で「このまま進める」「マッピングを修正したい」を確認する
4. **失敗時**: `AskUserQuestion` で各論理カラムに対応するヘッダーを選択させる（選択肢は全ヘッダー名）

## チーム構成ファイルの列構成

### 必須列

- **チーム名**: チームの識別子
- **学籍番号**: メンバーの学籍番号
- **氏名**: メンバーの氏名

1学生1行で、同じチーム名を持つ行のメンバーを同一チームに集約する。

### カラム名の揺らぎ対応

履修者名簿の揺らぎ対応に加えて以下を認識する:

| 論理カラム | 受理するヘッダー名 |
|-----------|-----------------|
| チーム名 | `チーム名`, `班`, `班名`, `グループ`, `グループ名`, `Team` |

### 例

```csv
チーム名,学籍番号,氏名
チームA,22001,山田太郎
チームA,22002,佐藤花子
チームA,22003,鈴木一郎
チームB,22004,田中次郎
チームB,22005,高橋三郎
```

## 未提出判定への利用

Step 3 で履修者名簿が提供された場合、名簿に含まれる学生のうち Step 3 の対応表推定で検出されなかった学生を「未提出」として記録する。名簿が提供されない場合は提出物から検出された学生のみを採点対象とし、未提出判定は行わない。
