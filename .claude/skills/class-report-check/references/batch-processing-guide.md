# バッチ処理ガイド

Step 3 のディレクトリスキャンと Step 5 の一括処理で参照する。

## ディレクトリ構造のパターン

### パターン1: 学生ごとのサブディレクトリ

```
submissions/
├── 22001_山田太郎/
│   ├── report.md
│   └── main.py
├── 22002_佐藤花子/
│   └── レポート1.pdf
└── 22003_鈴木一郎/
    └── report.md
```

サブディレクトリ名から学籍番号と氏名を抽出する。区切り文字は `_`（アンダースコア）を標準とし、`-`（ハイフン）やスペースにも対応する。

### パターン2: フラット配置

```
submissions/
├── 22001_山田太郎_report.md
├── 22002_佐藤花子_report.pdf
└── 22003_鈴木一郎_report.md
```

ファイル名から学籍番号と氏名を抽出する。同一学生の複数ファイルはファイル名のプレフィックスで紐付ける。

### パターンが不明な場合

自動判定できない場合はユーザーに確認する。ファイル名・ディレクトリ名の命名規則を教えてもらう。

## 対応ファイル形式

| 形式 | 拡張子 | 読み込み方法 |
|------|--------|-------------|
| Markdown | .md, .txt | Read ツール |
| Python | .py | Read ツール |
| JavaScript | .js, .ts | Read ツール |
| C/C++ | .c, .cpp, .h | Read ツール |
| Java | .java | Read ツール |
| Ruby | .rb | Read ツール |
| PDF | .pdf | Read ツール（pages パラメータ使用、20ページまで） |

## スキップするファイル

以下のファイルは自動的にスキップする:
- `.DS_Store`, `Thumbs.db`
- `__pycache__/`, `.git/`, `node_modules/`
- 画像単体ファイル（`.png`, `.jpg`, `.jpeg`, `.gif`, `.svg`）
- アーカイブファイル（`.zip`, `.tar.gz`）
- 実行ファイル（`.exe`, `.out`）

## 1提出者が複数ファイルを持つ場合

すべての対応ファイルを読み込み、以下の優先順で主ファイルを判定する:

1. **レポートファイル**: `.md` > `.pdf` > `.txt`
2. **コードファイル**: メインファイル（`main.py`, `index.js` 等）> その他のコードファイル

採点では全ファイルの内容を総合的に評価する。

## 読み込みエラーへの対応

- ファイルが読み込めない場合: スキップし、採点一覧に「読み込みエラー」と記録する
- PDFが20ページを超える場合: 最初の20ページのみ読み込み、その旨を記録する
- 空ファイルの場合: 「空ファイル」として記録し、「未到達」で採点する

## チーム提出時のディレクトリ構造

```
submissions/
├── チームA_山田_佐藤_鈴木/
│   ├── report.md
│   └── src/
│       └── main.py
└── チームB_田中_高橋_渡辺/
    └── report.pdf
```

チーム名とメンバー名はディレクトリ名または Step 0 で入力されたチーム構成情報から取得する。
