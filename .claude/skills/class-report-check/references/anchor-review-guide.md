# アンカー採点ガイド

Step 4 で参照するアンカー採点の手順。

## アンカー採点とは

全件採点の前に3件のサンプルを選んで先行採点し、ユーザーに確認を取ることで、残り全件の採点基準を確立する方式。これにより:

- 教員が納得した基準で全件を採点できる
- 採点のドリフト（途中で基準がずれる）を防止できる
- 「同じ品質のレポートには同じ点数」を実現できる

## サンプリング基準

サンプリングの目的は「**品質のバラツキを含む3件で基準を立てる**」こと。代理指標（記述量）は完璧でなくてよいが、測定方法は以下に固定する。

### 記述量の測定方法（ファイル形式別）

| 提出物の形式 | 測定値 |
|-------------|-------|
| Markdown / テキストレポート | 本文の文字数（図表・コードブロック内は除外） |
| PDF | テキスト抽出後の文字数 |
| ソースコード | 有効コード行数（コメント行・空行を除いた LOC） |
| プログラム + レポート（複合提出） | レポート本文の文字数を優先。レポートが無い場合はコードの有効 LOC |

### サンプリング手順

1. 全提出物について上記方法で記述量を測定する
2. 以下の基準で3件を選定する:
   - **最大値の1件**（高品質の見込み）
   - **最小値の1件**（低品質の見込み）
   - **中央値付近の1件**

### 3件が似た品質だった場合

代理指標だけでは判定しきれないため、選定後に実際に内容を読んで適切かを判断する。3件の品質が同程度だった場合、`AskUserQuestion` で以下のいずれかを選ばせる:

- options: 「このまま進める」「別の3件でやり直す」

「別の3件でやり直す」が選択された場合、除外された提出物から別の3件を再選定する。

### 提出件数が3件未満の場合

- **3件**: アンカー採点で全件処理できるため、一括採点ステップ（Step 5）はスキップする
- **2件以下**: サンプリングを行わず、全件を直接「アンカー採点」と同じ詳細採点で処理する。Step 4 と Step 5 を統合して扱う

## アンカー採点の実施

3件それぞれに対して `scoring-guide.md` の採点手順を適用する。結果を以下の形式でユーザーに表示する:

```
=== アンカー採点: [学籍番号] [氏名] ===
（記述量: [N]字 / ファイル: [ファイル名]）

### [観点1名]
評価: [段階]
点数: [N]点 / [M]点
根拠: [scoring-guide.md の引用フォーマットに従う]

### [観点2名]
...

---
総合点: [N]点 / 100点
```

3件すべてのアンカー採点を提示してから、ユーザーに確認を取る。

## アンカー確定後の参照方法

ユーザーがアンカー採点を承認した後、Step 5 の一括採点で以下のように参照する:

- 各提出物を読んだ後、まず「アンカーAより良い / 同程度 / 劣る」を各観点で判断する
- その判断をルーブリックの段階判定の補助情報として使う
- ルーブリック照合結果とアンカー比較結果が矛盾する場合は、ルーブリック照合を優先する

## アンカー修正時の再採点

ユーザーがアンカー採点を修正した場合:
- 修正内容がルーブリック解釈の変更を伴う場合、他の2件のアンカーも再確認する
- 一括採点がまだ開始されていなければ、修正後のアンカーで一括採点を開始する
