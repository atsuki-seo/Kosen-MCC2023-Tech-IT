# アンカー採点ガイド

Step 4 で参照するアンカー採点の手順。

## アンカー採点とは

全件採点の前に3件のサンプルを選んで先行採点し、ユーザーに確認を取ることで、残り全件の採点基準を確立する方式。これにより:

- 教員が納得した基準で全件を採点できる
- 採点のドリフト（途中で基準がずれる）を防止できる
- 「同じ品質のレポートには同じ点数」を実現できる

## サンプリング基準

提出物ディレクトリをスキャンした後、以下の基準で3件を選定する:

### 1. 最良候補（高品質の見込み）
- ファイルサイズ・記述量が最も多い提出物
- 複数ファイルを提出している場合は優先的に選ぶ

### 2. 最劣候補（低品質の見込み）
- ファイルサイズ・記述量が最も少ない提出物
- 1ファイルのみの提出を優先的に選ぶ

### 3. 中程度候補
- ファイルサイズ・記述量が中央値付近の提出物

ファイルサイズだけでは品質を正確に推測できないため、選定後に実際に内容を読んで適切かを判断する。3件がすべて同程度の品質だった場合、ユーザーに「別の3件でやり直す」オプションを提示する。

## アンカー採点の実施

3件それぞれに対して `scoring-guide.md` の採点手順を適用する。結果を以下の形式でユーザーに表示する:

```
=== アンカー採点: [学籍番号] [氏名] ===
（記述量: [N]字 / ファイル: [ファイル名]）

### [観点1名]
評価: [段階]
点数: [N]点 / [M]点
根拠: [scoring-guide.md の引用フォーマットに従う]

### [観点2名]
...

---
総合点: [N]点 / 100点
```

3件すべてのアンカー採点を提示してから、ユーザーに確認を取る。

## アンカー確定後の参照方法

ユーザーがアンカー採点を承認した後、Step 5 の一括採点で以下のように参照する:

- 各提出物を読んだ後、まず「アンカーAより良い / 同程度 / 劣る」を各観点で判断する
- その判断をルーブリックの段階判定の補助情報として使う
- ルーブリック照合結果とアンカー比較結果が矛盾する場合は、ルーブリック照合を優先する

## アンカー修正時の再採点

ユーザーがアンカー採点を修正した場合:
- 修正内容がルーブリック解釈の変更を伴う場合、他の2件のアンカーも再確認する
- 一括採点がまだ開始されていなければ、修正後のアンカーで一括採点を開始する
