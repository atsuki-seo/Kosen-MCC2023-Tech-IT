# Git ワークフロー（プロジェクト固有ルール）

このファイルはグローバルルール `~/.claude/rules/git-workflow.md` を本プロジェクトで上書きする。

## main への直接コミット・プッシュ

本プロジェクトでは `main` への直接コミット・プッシュを**許可する**。

- グローバルルールの「保護ブランチには直接コミット・プッシュしない」は**本プロジェクトでは適用しない**
- 作業ブランチ・PR 経由のフローは必須ではない（ユーザーが PR での対応を指示した場合のみ従う）
- ユーザーから `main` への push 指示があった場合は、確認を挟まず実行してよい

## 保護ブランチでの作業

- worktree の使用は作業の切り分けに有用な場合に推奨するが、**必須ではない**
- 差分が未コミットのまま別作業に移る等の副作用回避のため worktree が有効だと判断した場合は使ってよい
- ユーザーの指示で現在のブランチで直接作業することも認める

## 引き続き禁止する操作

以下は本プロジェクトでも引き続き禁止する:

- force push（`--force` / `--force-with-lease` ともユーザーの明示指示がなければ使わない）
- `git reset --hard` など破壊的コマンドのユーザー承認なしの実行
- hook のスキップ（`--no-verify` 等）

## Claude 設定ファイルの扱い

グローバルルールの「プロジェクトスコープの Claude 設定ファイル（`.claude/skills/`, `.claude/rules/`, `CLAUDE.md` 等）は、ユーザーから明示的に指示があるまで git の差分に含めない」は**引き続き有効**。これらのファイルへの変更作業自体がユーザー指示で行われている場合（本ルールの新設等）は、その作業の対象ファイルを差分に含めてよい。
