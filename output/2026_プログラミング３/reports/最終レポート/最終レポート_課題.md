## 課題の目的
第8〜14週のチーム開発プロジェクトを通じて、到達目標「git/GitHubを用いたバージョン管理とチーム開発の実践」および「チームでのソフトウェア開発プロジェクトにおける協調作業と工程管理」の達成度を評価する。チームで1つのシステムを企画・設計・実装し、成果物として公開する。

## 課題の内容

### テーマ
チームで企画したシステムを、git/GitHub を用いたチーム開発で実装・公開する。テーマは第8週のチーム編成時に決定し、教員の確認を得ること。

### 要件

**必須要件**
- GitHub 上にチームリポジトリを作成し、成果物（ソースコード）を公開すること
- 全メンバーがソースコード作成・Issue 作成・Pull Request 作成のいずれかに関わること（`git log --author` と GitHub のコントリビューション履歴で確認できる状態にすること）
- 以下の仕様書を作成し、リポジトリ内（および提出ディレクトリ）に配置すること:
  - 要件定義書（第9週に着手、第14週時点で最終版）
  - 基本設計書（画面設計・DB設計・システム構成）
  - 詳細設計書（処理フロー・モジュール設計）
- ブランチ戦略を採用し、main への直接 push ではなく Pull Request 経由でマージすること（最低3件以上の PR ログを残すこと）
- Issue で作業管理を行うこと（最低10件以上の Issue ログを残すこと）
- 成果物の動作手順を README.md にまとめること
- 各メンバーの担当箇所または貢献内容を振り返り報告書に明記すること
- 生成AI活用のプロンプト記録をチームで集約して提出すること
- KPT形式の自己評価（第9〜14週分）をチームとしてまとめ、振り返り報告を作成すること

**推奨要件**
- CI/CD の導入（GitHub Actions 等）
- コードレビューのコメント履歴の充実

### 取り組みの手順
1. 第8週: チーム編成・テーマ決定・GitHubリポジトリ作成
2. 第9週: 要件定義書作成、Issue による作業管理開始、第1回KPT提出
3. 第10週: 基本設計書作成、設計内容に基づくIssue作成
4. 第11週: 詳細設計書作成、Issue/PRを活用した本格的開発開始
5. 第12週: 実装、PRによるコードレビュー実施、AI活用プロンプト記録開始
6. 第13週: 実装継続・動作検証、AI活用記録共有
7. 第14週: 成果物最終調整、仕様書PDF化、振り返り報告作成、提出

## 提出物の仕様
- 提出物一式:
  - GitHubリポジトリURL（公開設定、または教員アカウントを招待）
  - 要件定義書・基本設計書・詳細設計書（PDF形式、リポジトリ内にも配置）
  - チーム振り返り報告書（Markdown、KPT集約と担当貢献を含む）
  - AI活用プロンプト記録（Markdown）
- 提出ディレクトリ名: `チーム名_メンバー氏名1_メンバー氏名2_..`（例: `チームA_山田_佐藤_鈴木`）
- 提出先: Moodle（科目ページの「最終レポート提出」課題）にディレクトリを ZIP 圧縮してアップロード
- チームメンバー全員の氏名・学籍番号を振り返り報告書の冒頭に記載すること

## 評価基準
| 評価観点 | 配点 | 概要 |
|----------|------|------|
| git/GitHub の活用 | 25点 | コミット履歴・ブランチ運用・PR件数とレビューの質 |
| Issue/PR によるチーム開発運営 | 20点 | Issue数と作業管理状況、PR/レビューの運用 |
| 仕様書の網羅性と論理性 | 20点 | 要件定義・基本設計・詳細設計の3点セットの完成度 |
| 成果物の完成度 | 15点 | README に従って動作するか、計画機能の実装状況 |
| チーム協働・役割分担 | 10点 | 全員の貢献記録、担当箇所の明示 |
| AI活用記録と振り返り | 10点 | プロンプト記録の網羅性、KPT振り返りの質 |
| **合計** | **100点** | — |

## チーム提出に関する注意事項

- **提出ディレクトリ名**: `チーム名_メンバー氏名1_メンバー氏名2_..`（例: `チームA_山田_佐藤_鈴木`）
- **提出ファイル構成**: ディレクトリ内に `README.md`・`振り返り報告書.md`・`AI活用記録.md`・仕様書PDF3点・リポジトリへのリンクをまとめた `提出情報.md` を配置する
- チームメンバー全員の氏名・学籍番号を振り返り報告書の冒頭に記載すること
- 各メンバーの担当箇所または貢献内容を明記すること（例: 「要件定義: 山田」「バックエンド実装: 佐藤」「フロント実装: 鈴木」等）
- 全メンバーがソースコード作成・Issue作成・PR作成のいずれかに関わっていることが git/GitHub の履歴から確認できる状態にすること
