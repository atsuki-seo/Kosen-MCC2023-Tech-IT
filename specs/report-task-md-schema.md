# レポート課題MD・ルーブリックMD スキーマ

`class-report` が出力し、`class-report-check` が読み取る `[レポート名]_課題.md` および `[レポート名]_ルーブリック.md` の**入出力契約**。

このファイルは「出力仕様（class-report 生成時のルール）」と「入力仕様（class-report-check 抽出時のルール）」を一つに束ねた**単一の正本**である。各スキルのリファレンスに同じ定義を二重記述せず、必ず本ファイルを参照すること。

## 参照元スキル

- `class-report`: Step 7 のファイル出力で本スキーマに従って生成する
- `class-report-check`: Step 1 のメタデータ読み取りで本スキーマに従って抽出する
- `class-slides`（任意参照）: 該当回のスライドに「今回の提出物」を展開する際に基本情報・課題内容を参照する

## 関連スキーマ

- ルーブリックの段階構造・点数換算: `specs/rubric-schema.md`
- ファイル配置パス: `specs/output-layout.md`
- チーム構成CSV: `specs/roster-team-csv.md`

## 1. 課題MD（`[レポート名]_課題.md`）

### 1.1 全体構造

```markdown
# [レポート名]: [科目名]

## 基本情報
（表形式）

## 課題の目的
（本文）

## 課題の内容

### テーマ
（本文）

### 要件
（必須要件・推奨要件の箇条書き）

### 取り組みの手順
（番号付き箇条書き）

## 提出物の仕様
（箇条書き）

## 評価基準
（表形式）

## チーム提出に関する注意事項
（チーム提出の場合のみ）
```

### 1.2 基本情報テーブル

`## 基本情報` セクションで以下を表形式で出力する。`class-report-check` は本テーブルからメタデータを抽出する。

| 項目 | 必須 | 備考 |
|------|------|------|
| 科目名 | ○ | |
| 担当教員 | ○ | |
| 対象学年 | ○ | |
| レポート名 | ○ | `syllabus-markdown-schema.md` § 5.1 で正規化済みの名前 |
| 提出期限 | ○ | `第[N]週` 形式 |
| 提出形態 | ○ | `個人` または `チーム` |
| 提出形式 | ○ | `Markdown / テキスト` `PDF` `プログラム（ソースコード）` `プログラム + レポート` のいずれか |

### 1.3 提出物の仕様

`## 提出物の仕様` セクションには以下を含める:

- ファイル形式
- 命名規則: `学籍番号_氏名_レポート名.拡張子` を標準とする（例: `22001_山田太郎_レポート1.md`）
- 提出先
- 分量の目安

### 1.4 評価基準テーブル

`## 評価基準` セクションで以下を表形式で出力する。詳細なルーブリック表は `[レポート名]_ルーブリック.md` 側に記載するため、ここでは観点名・配点・1行説明のみを出力する。

```markdown
| 評価観点 | 配点 | 概要 |
|----------|------|------|
| [観点1] | [N]点 | [1行の説明] |
| [観点2] | [N]点 | [1行の説明] |
| ... | ... | ... |
| **合計** | **100点** | — |
```

合計は必ず100点。配点合計が100にならない場合は契約違反。

### 1.5 チーム提出時の追加セクション

`## 提出物の仕様` の後に `## チーム提出に関する注意事項` セクションを追加する。

```markdown
## チーム提出に関する注意事項

- **提出ディレクトリ名**: `チーム名_メンバー氏名1_メンバー氏名2_メンバー氏名3`（例: `チームA_山田_佐藤_鈴木`）
- **提出ファイル名**: ディレクトリ内に `report.md` 等のファイルを配置する
- チームメンバー全員の氏名・学籍番号をレポート冒頭に記載すること
- 各メンバーの担当箇所または貢献内容を明記すること（例: 「第2章: 山田太郎」、「実装担当: 山田、検証担当: 佐藤」等）
```

課題の性質に応じて、担当箇所の明記が必要か・貢献内容の記載が適切かを判断し、文言を調整してよい。

## 2. ルーブリックMD（`[レポート名]_ルーブリック.md`）

### 2.1 全体構造

```markdown
# ルーブリック: [レポート名]（[科目名]）

## 基本情報
（表形式）

## 点数換算基準
（表形式・任意。デフォルトと異なる場合に出力）

## ルーブリック
（表形式）

## 採点メモ欄
（表形式・教員手動採点用テンプレート）
```

### 2.2 基本情報テーブル

| 項目 | 必須 | 備考 |
|------|------|------|
| 科目名 | ○ | |
| レポート名 | ○ | |
| 配点 | ○ | `100点満点` 固定 |
| 提出形態 | ○ | `個人` または `チーム` |

### 2.3 点数換算基準

`specs/rubric-schema.md` § 2 のデフォルト（理想的=80〜100%/標準点90%、標準的=60〜79%/標準点70%、未到達=0〜59%/標準点30%）と異なる場合に限り、本セクションを出力する。

```markdown
## 点数換算基準

| 段階 | 点数範囲（観点配点に対する比率） |
|------|-------------------------------|
| 理想的 | 80〜100%（標準点: 90%） |
| 標準的 | 60〜79%（標準点: 70%） |
| 未到達 | 0〜59%（標準点: 30%） |
```

`class-report-check` は本セクションが存在する場合は優先して使用し、無い場合は `specs/rubric-schema.md` § 2 のデフォルトを適用する。

### 2.4 ルーブリック表

`specs/rubric-schema.md` § 1.2 の形式で出力する。

```markdown
## ルーブリック

| 評価観点 | 理想的 | 標準的 | 未到達 | 配点 |
|----------|--------|--------|--------|------|
| [観点1] | [理想的の記述] | [標準的の記述] | [未到達の記述] | [N]点 |
| ... | ... | ... | ... | ... |
| **合計** | — | — | — | **100点** |
```

### 2.5 採点メモ欄

教員が手動採点する際のテンプレート。`class-report-check` の自動採点では**無視する**。

```markdown
## 採点メモ欄

| 評価観点 | 評価 | 点数 | メモ |
|----------|------|------|------|
| [観点1] | □理想的 □標準的 □未到達 | /[N] | |
| [観点2] | □理想的 □標準的 □未到達 | /[N] | |
| ... | ... | ... | |
| **合計** | — | **/100** | |
```

## 3. 抽出仕様（class-report-check 側）

`class-report-check` の Step 1 では、課題MDから以下のメタデータを抽出する。

| 表示項目 | 抽出元 |
|---------|-------|
| 科目名 | `## 基本情報` テーブルの「科目名」行 |
| レポート名 | `## 基本情報` テーブルの「レポート名」行 |
| 提出形態（個人/チーム） | `## 基本情報` テーブルの「提出形態」行 |
| 課題の概要 | `## 課題の目的` セクション |
| 提出物の要件 | `## 提出物の仕様` セクション |
| 評価観点 | `## 評価基準` テーブル（詳細はルーブリックMDから取得） |

ルーブリックMDからは以下を抽出する:

| 表示項目 | 抽出元 |
|---------|-------|
| ルーブリック表 | `## ルーブリック` テーブル |
| 点数換算基準 | `## 点数換算基準` テーブル（無ければ `specs/rubric-schema.md` § 2 のデフォルト） |

`## 基本情報` テーブルが存在しない（手動作成された課題）場合、提出形態のみ `AskUserQuestion` で確認する。

---

## 4. フォーマット規則

1. **セクション順序**: 課題MDは「基本情報 → 目的 → 内容 → 提出物 → 評価基準 → チーム注意事項」、ルーブリックMDは「基本情報 → 点数換算基準 → ルーブリック → 採点メモ欄」の順で固定
2. **表形式**: Markdownテーブルを使用、列の揃えは左寄せ
3. **合計行**: 配点テーブルには必ず合計行を含める
4. **エンコーディング**: UTF-8、BOMなし
